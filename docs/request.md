# 《计算机系统结构》项目制教学改革方案(初稿) 
## 一、总体思路 
• 《计算机系统结构》：造一台 “虚拟计算机” 
以指令集架构 + CPU 微结构 + 硬件抽象为核心，面向 “硬件设计者” 视角。使用高级编程语言（C/C++、Rust等），以指令集模拟器为核心目标，完成 CPU、指令、内存、中断、外设的模拟实现。支持AI辅助。 
→ 输出：可独立运行的虚拟机 /myCPU 模拟器。 

• （下阶段可选）《操作系统》：在虚拟机上跑一个 “完整操作系统” 以程序加载、系统启动、最小 OS、LFS 系统构建为核心项目。 
→ 输出：可在模拟器上运行的 miniOS / Linux 最小系统。

• （下阶段可选、加分）联动模式：系统结构课输出的模拟器作为硬件平台 操作系统课输出的OS 镜像在该平台上运行 
→ 形成：自己写 CPU → 自己写 OS → 自己跑 Linux的完整闭环。 

## 二、项目技术方案 
### 1. 项目名称 
myCPU：从 0 实现一个可运行程序的指令集模拟器 
对标Bochs模拟器，选择的架构：RISC-V 
### 2. 课程目标 
1) 理解指令集架构（ISA）与 CPU 执行模型。 
2) 掌握取指–译码–执行的流水线思想。 
3) 理解内存、寄存器、异常、中断、外设接口原理。 
4) 能够用高级语言实现一个可运行裸机程序的模拟器。 
### 3. 项目内容 
1) CPU 核心模块 
    o 寄存器组、PC、标志位设计 
    o 基础指令集实现（30～40 条） 
    o 流水线主循环 
2) 内存与地址空间模块： 
    o 物理内存模拟 
    o 地址检查、越界异常 
    o MMU / 分页机制（基础版） 
3) 中断与异常模块（加分待定） 
    o 中断控制器模拟 
    o 异常入口、上下文保存与恢复 
    o 时钟中断（为操作系统预留） 
4) 外设接口模块（加分待定） 
    o UART 串口（打印输出） 
    o 定时器中断 
    o 统一硬件接口规范（便于后续对接 OS） 
### 4. 项目成果 
• 模拟器可运行，可串口输出、处理中断、执行运算 

• 完整技术报告、演示视频
